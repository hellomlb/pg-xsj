CVE-2026-6473：lquery 整数溢出导致服务端崩溃与静默错误

2026-05-25 CVE 验证

PG 安全发布修复了一个 8.8 分整数溢出漏洞，影响 14~18 所有版本。本文通过 PG 16.11 vs 16.14 双版本实测验证：totallen 溢出直接导致服务端崩溃，numvar 溢出静默返回错误结果——两种均为可被未授权用户触发的真实安全威胁。

PostgreSQL Bug 深度分析与修复解读

Bug #17：MERGE 在 RR/SERIALIZABLE 下静默违反隔离性

2026-05-25 Bug 分析

PG 16.14 修复了一个隐蔽的并发 Bug：MERGE 在 REPEATABLE READ / SERIALIZABLE 隔离级别下遇到并发更新时，静默使用新版本元组继续操作，不报序列化失败。16.11 就此漏过，应用层根本不知道数据已经错了。
