escape_string_warning：守了二十年的 SQL 注入哨兵，在 PG19 谢幕

2026-07-15 安全加固

escape_string_warning 是 PostgreSQL 默认开启的 GUC，检测应用是否偷偷依赖非标准字符串转义（SQL 注入防护的检测端）。PG 16.11 + 16.14 双版本实测其完整行为，并揭示 PG 19 Beta1 已将其移除、彻底禁用非标准字符串字面量——老应用 'O\'Brien' 写法升级到 19 会直接报错。
