DeepSeek RAG 本地知识库 — 安装部署文档

2026-05-29 AI DeepSeek RAG ChromaDB fastembed 本地知识库

架构：fastembed（本地向量化） + ChromaDB（向量存储） + DeepSeek API（对话生成）
文档隐私：原始文档不上传云端，仅对话时发送检索到的片段给 DeepSeek API
适用配置：8GB 内存 + 任意 CPU（实测华为 MateBook 14, i5-1155G7 运行流畅）

1. 系统要求

项目最低配置推荐配置
内存4GB8GB+
磁盘5GB 可用10GB+
操作系统Windows 10 / Linux / macOS任意
Python3.8+3.13（实测通过）
网络需要连接 huggingface 镜像（首次下载 45MB 模型）国内网络需改镜像

3. 安装步骤

3.1 安装 Python 3.8+

确保 python --version 输出 3.8 以上。

3.2 创建虚拟环境并安装依赖

# Windows
python -m venv .venv
.venv\Scripts\activate

# Linux / macOS
python3 -m venv .venv
source .venv/bin/activate

# 安装依赖（建议使用国内镜像加速）
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple \
    openai chromadb langchain langchain-community langchain-openai \
    langchain-core pypdf python-docx tiktoken streamlit fastembed chardet langchain_text_splitters
包名用途安装大小
fastembed本地向量化（替代 OpenAI Embeddings）~50MB
chromadb向量数据库~50MB
langchainRAG 流程编排~10MB
langchain-openaiDeepSeek Chat API 调用~1MB
streamlitWeb 界面~50MB
pypdfPDF 文档加载~5MB
python-docxWord 文档加载~5MB
chardet文本编码检测~5MB
onnxruntimefastembed 的推理引擎（自动安装）~30MB

4. 配置 DeepSeek API

4.1 注册并获取 Key

  1. 打开 platform.deepseek.com
  2. 注册账号 → 在 "API Keys" 创建 Key → 复制保存
  3. 进入"充值"页面至少充值 ¥10（对话模型 ¥2/百万 token，非常便宜）

4.2 填入配置

编辑 config.py：

# DeepSeek API（仅用于对话生成）
DEEPSEEK_API_KEY = "sk-your-key-here"     # ← 你的 API Key
DEEPSEEK_BASE_URL = "https://api.deepseek.com/v1"
CHAT_MODEL = "deepseek-v4-flash"          # 或 deepseek-v4-pro

# 本地路径（按需修改）
DOCS_DIR = r"D:\AI知识库\docs"            # 你的文档目录
CHROMA_DIR = r"C:\Users\用户名\.workbuddy\chroma_db"

# 文本切分参数（一般不用改）
CHUNK_SIZE = 500
CHUNK_OVERLAP = 50

# 检索参数
TOP_K = 4  # 每次检索返回的相似片段数

4.3 重要提醒

  • DeepSeek 没有 Embedding API！向量化由本地的 fastembed 完成（免费、隐私）
  • Embedding 模型自动从 hf-mirror.com 下载（约 45MB），无需手动操作
  • 只有对话环节才会调用 DeepSeek API，产生费用

5. 准备文档

支持的格式：

格式后缀说明
Markdown.md推荐，原生支持
纯文本.txt编码需为 UTF-8
PDF.pdf需安装 pypdf（已包含）
Word.docx需安装 docx2txt（可选）

6. 构建向量库（首次运行）

# 确保虚拟环境已激活
.venv\Scripts\activate

# 构建向量库
python cli.py build

构建时间参考：72 个文档约 3~10 分钟（取决于机器性能）。

后续新增文档：python cli.py add（增量，不重复处理已有文档）。

查看状态：python cli.py status

7. 启动问答

Web 界面（推荐）

.venv\Scripts\streamlit run web_ui.py

浏览器自动打开 http://localhost:8501，左侧栏可管理知识库，主区域对话并显示引用来源。

命令行交互

python cli.py ask

9. 常见问题

Q1: DeepSeek Embedding API 返回 404

DeepSeek V4 没有 Embedding API 端点。已改用本地 fastembed，开箱即用。

Q2: 模型下载失败（HuggingFace 被墙）

代码自动设置了 HF_ENDPOINT=https://hf-mirror.com，如果未生效手动设置：

# Windows PowerShell
$env:HF_ENDPOINT="https://hf-mirror.com"

# Linux / macOS
export HF_ENDPOINT=https://hf-mirror.com

Q3: ONNX Runtime 内存不足

fastembed 默认批量处理，8GB 机器可能不够。已内置 batch_size=32，如仍报错可改为 16。

Q4: DeepSeek API 返回 402

账号余额不足，去 platform.deepseek.com 充值。

国内镜像hf-mirror.com解决 HuggingFace 被墙问题

为什么不用 DeepSeek Embedding？

实测 DeepSeek API 的 /embeddings 端点返回 404，其官方模型列表仅包含对话模型，无 Embedding 模型。因此采用本地 fastembed 替代，优点：

  • 完全免费，不限次数
  • 文档不上传任何云端
  • 45MB 模型，8GB 机器流畅运行
