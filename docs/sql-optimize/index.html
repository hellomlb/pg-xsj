enable_partitionwise_aggregate：分区聚合的内存放大与性能权衡

2026-07-07 SQL 优化

enable_partitionwise_aggregate 是 GUC 家族的异类——默认 off，因为开了会吃 N 倍内存。8 分区 80 万行双版本实测：GROUP BY 分区键时 ON 快 9%，GROUP BY 非分区键时快 22%，但小数据量反而慢 8%。内存放大公式 N × work_mem，100 分区 + 64MB = 6.4GB 一个查询。OLAP 角色级开启，OLTP 保持 off。

enable_partitionwise_join：分区对 Join 的性能提升与 M:N 爆炸陷阱

2026-07-08 SQL 优化

enable_partitionwise_join 默认 off，开启后把两个分区表的大 Join 拆成按分区对的 N 个小 Join，每对独立选最优算法。前提严格（两表按同一分区键分区 + 连接条件包含全部分区键），且有一个致命陷阱：只按分区键 Join 会触发 M:N 爆炸，产生数亿行中间结果。所有 SQL 在 PG 16.11 + 16.14 双版本实测。
